## Capture Workflow

### Step 1: Context Gathering (~5s)

**1.1 — Read the target skill**
Read `<SkillRoot>/<skillName>/SKILL.md` in full. This is the authoritative current state.

**1.2 — Inspect skill structure**
List files in `<SkillRoot>/<skillName>/references/` to understand what reference files exist.
This informs `Target file` selection in Step 2.

**1.3 — Parse learningText**
`learningText` = all tokens after `skillName` in `$ARGUMENTS`. Treat as freeform prose — the
user's raw observation from the current session.

---

### Step 2: Improvement Formulation (~15s)

**2.1 — Synthesize the improvement**
Using this session's conversation context plus the skill text from Step 1, formulate a
structured improvement. The user provides direction; infer the specific improvement from
context. Do not invent problems not evidenced by the session.

**2.2 — Extract Session evidence**
Pull the specific observable event from this session that surfaces the learning. Aim for
2-4 sentences covering: what the skill did (or failed to do), what the correct behavior
should be, and any error messages or missed steps observed. Must be self-contained for a
reader with no session context.

**2.3 — Determine Target file**
- Main flow, entry point, or constraints → `SKILL.md`
- Specific phase or workflow step → `references/<matching-file>.md`
- No single reference file owns the gap → `SKILL.md`

Never propose creating a new reference file; flag it for human decision instead.

**2.4 — Assess confidence**
- **high** — clear gap, specific fix, strong session evidence
- **medium** — gap exists but fix approach is uncertain
- **low** — loose observation; capture anyway for review

**2.5 — Duplicate check**
If `improvements/plan.md` exists, scan all `### IMP-` entries. If any targets the same
`Target file` and describes a substantially similar change, skip and output:
```
This is similar to IMP-<N>. Skipping.
```

---

### Step 3: Append to Plan (~2s)

**3.1 — Ensure improvements directory**
Create `<SkillRoot>/<skillName>/improvements/` if it does not exist.

**3.2 — Ensure plan.md exists**
If `improvements/plan.md` does not exist, create it with this exact header:
```markdown
# Improvement Plan: <skillName>

> Auto-generated by /improve-skill. Do not edit manually unless you know what you're doing.
```

**3.3 — Determine next IMP number**
Count existing `### IMP-` headings in `plan.md` and add 1. New file → start at 1.

**3.4 — Append the entry**
Add one blank line before the new heading, then append:
```markdown
### IMP-<N>: <title>
**Captured**: <YYYY-MM-DD>
**Skill**: <skillName>
**Target file**: <SKILL.md | references/phase-X.md>
**Session evidence**: <what happened — concise but self-contained for cross-session use>
**Current state**: <quote or summary of the relevant skill text as it exists now>
**Proposed change**: <what should change and why it's better>
**Confidence**: <high | medium | low>
```

Use today's date (`YYYY-MM-DD`) for `Captured`. `<total>` = total `### IMP-` heading count after appending.

**3.5 — Confirm capture**
Output exactly:
```
Captured IMP-<N> for <skillName>. <total> improvements pending.
```
